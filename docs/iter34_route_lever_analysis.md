> 触发：Iter 32.5 真实基线 N=67 群体归因——quality-route 桶（N=30）胜率最低 0.43，我方鲜度 82.6 vs 对手 93.2（gap +10.5），roadmap 据此排「Iter34+ 静态最优：quality-route 鲜度杠杆 = Phase B 静态规划器路线重选」。

## 1. 路线杠杆证伪

### 1.1 真实地图拓扑（取自 `reports/` quality-route 败局 compact.log Map 行）

```
N S01:S S02:C S03:P S04:D S05:W S06:M S07:S S08:M S09:S S10:K S11:P S12:J S13:P S14:G S15:F
E S01-S02:30:R S02-S03:25:R S03-S07:54:R S07-S09:46:R S09-S10:40:R S10-S11:36:R

  S07-S08:42:M S04-S07:54:B S08-S09:64:B S11-S14:15:B S10-S13:27:B
```

### 1.2 三准则最优路（Dijkstra，权重独立）

边权公式（`core/rules.py`）：
- 帧数 `frames = ceil(ceil(distance × ROUTE_TIME_COEF[rt]) / 1000)`，`ROUTE_TIME_COEF={ROAD:1380,WATER:1250,MOUNTAIN:1780,BRANCH:1550}`
- 鲜度损耗 `loss = frames × FRESHNESS_LOSS_MOVE[rt]`，`{ROAD:0.055,WATER:0.045,MOUNTAIN:0.07,BRANCH:0.065}`
- 分数成本（边际）`= frames × (用时/frame + 鲜度损耗/frame × 1.8)`，`用时/frame ≈ 70/600 = 0.117`，`鲜度分/freshness = 180/100 = 1.8`

| 准则 | 最短路 | 总帧 | 鲜度损耗 |
|---|---|---|---|
| 时间最优（min frames） | S01→S02→S04→S05→S09→S10→S13→S14→S15 | 322 | 16.98 |
| 鲜度最优（min loss） | 同上 | 322 | 16.98 |
| 分数最优（min score-cost） | 同上 | 322 | 16.98 |

**三准则完全重合**。时间最优路已走 WATER 边（S04-S05、S05-S09）——WATER 同时是 `ROUTE_TIME_COEF` 最低（最快）与 `FRESHNESS_LOSS_MOVE` 最低（最保鲜）的路线类型。**路线选择在本图不是鲜度杠杆**：任何"绕更长的低损耗路"都不存在——已是最优。

### 1.3 对 roadmap 的修正

Phase B 静态规划器（`strategy/static_planner.py`，`ENABLE_STATIC_PLANNER` 默认关）的核心是**鲜度感知路线重选**。§1.2 证伪其前提：本图无更优路线可选，规划器在真实地图上是 no-op。Iter 26–28 sim A/B 中性（mean +0.1）的真正原因不仅是"samples 无廉价鲜度"，而是**真实图本身也无廉价鲜度路线**。**静态规划器是 quality-route 鲜度 gap 的错误工具，Iter 34 不开启，后续不再为其投入 A/B。**

## 2. 真正根因：鲜度 gap 是操作层的，不是路线层的

### 2.1 同路、同天气，损耗差一倍

取代表性 quality-route 败局 `match_..._2696_vs_2986`：

| | 我方 | 对手 |
|---|---|---|
| 路线 | S01→…→S15（时间最优，唯一） | 同 |
| 交付帧 | 411 | 436（+25） |
| 交付鲜度 | 83.36 | 96.21 |
| MIN 鲜度 | **77.26** | **85.28** |
| 冰鉴使用 | 1 次 @帧319（用前 77.26） | 1 次 @帧244（用前 ~85） |
| 好果 | 97（3 坏） | 99（1 坏） |
| 鲜度分 | 150 | 173 |
| 总分 | 766 | 790（−24） |

鲜度端值公式（任务书 §3.3.1：冰鉴 +10、好果转坏不可逆）：
`end_freshness = 100 − total_loss + 10 × ice_uses`，**与冰鉴时机无关**。

- 我方 total_loss = 100 − 83.36 + 10 = **26.64**
- 对手 total_loss = 100 − 96.21 + 10 = **13.79**

同路同天气，对手损耗是我方的一半。**gap 不在路线**（§1 已证），而在**冰鉴使用时机**衍生出的 MIN 鲜度差异：我方在 77.26 才用冰鉴（已跌破 80），对手在 ~85 用（80 之上）。

### 2.2 为什么时机影响好果但不影响端鲜度

冰鉴 +10 一次性加到当前鲜度，端鲜度 = 100 − loss + 10（恒定）。但**好果转坏阈值（90/80/70/…）首次跌破各触发 1 篓**，不可逆。冰鉴把鲜度抬回去，已触发的转坏不恢复，但能阻止**后续**首次跌破。

- 阈值 78（原）：鲜度跌到 77.26 才用冰鉴 → 已跌破 90、80 → 2 坏；冰鉴只护住 70（而本图端鲜度 83 ≥ 70，70 本不会触底）→ **冰鉴浪费**。
- 阈值 81（新）：鲜度跌到 80.x 即用 → 90 已跌但 80 未跌 → 1 坏；冰鉴把鲜度抬到 90.x，后续衰减到 83 不再跌破 80 → **救回 1 篓好果**。

对手 MIN 85.28（>80）正是此机制——对手在 80 之上用冰鉴，只跌 90 一次。

## 3. 修复：`ICE_BOX_USE_BELOW` 78 → 81

### 3.1 弱优于证明（任务书规则下）

记端鲜度 F = 100 − L + 10（L = 路线总损耗，与时机无关），好果转坏阈值 90/80/70/…：

| 路线损耗区间 | F 范围 | 78-阈值坏果 | 81-阈值坏果 | 差 |
|---|---|---|---|---|
| 低损耗（不跌 80） | F ≥ 90 | 1（90） | 1（90） | 0 |
| 中损耗（跌 80 不跌 70，端≥80） | 80 ≤ F < 90 | 2（90,80） | **1（90）** | **−1（81 省）** |
| 高损耗（跌 70） | F < 80 | k | k | 0 |

- 中损耗区间：78 在跌 80 后才用冰鉴（80 已转坏），81 在跌 80 前用（80 不转坏）。**81 严格省 1 篓**。
- 其余区间：80 是否转坏由 F 是否 ≥80 决定，与时机无关，两者坏果数相同。

**结论：81 弱优于 78——永不更差，在端鲜度 ∈ [80,90) 的路线（真实图正是 F=83.36）严格省 1 好果（+1.8 分）。** 与 Iter 25/29/33 同属"弱优于/bug-fix 纪律下无条件合入"，非阈值赌博。

### 3.2 为什么不直接用 91（护 90）

91-阈值（`STATIC_PLANNER_ICE_USE_BELOW`）在 F < 90 的路线上无法救 90（端鲜度 <90，90 终会被跌），与 81 等价；但 91 在低损耗路线（F ≥ 90）会过早触发、把冰鉴用在一个本不需要救的阈值上，浪费库存。81 仅在真正需要时（鲜度实跌向 80）触发，更保守。故选 81 而非 91。

### 3.3 验收

- **单测**：`tests/test_economy.py` 新增 `test_ice_box_fires_just_above_80_threshold`（freshness=80.5 触发）+ `test_no_ice_box_at_82`（82 不触发）。client 285 全过（283+2）。
- **sim 50 种子回归门**：1.000 交付 / 0 STUCK / 0 对账误差 / mean 747.8（=Iter33 基线，零回归）。
  - sim 中性是 no-op 而非无效：sim 鲜度损耗更高（端鲜度 ~75.6 < 80，见 CLAUDE.md §4.4 残留偏差），80 终被跌穿、81 与 78 等价。**收益仅在真实图（端鲜度 83 ≥ 80）显现**——这正是 sim 无法验证、须真实 A/B 的部分，但弱优于性质保证不劣化。

## 4. 残留 gap 与后续（Iter 35+ 真实 A/B）

## 勘误（Iter 34 合入后补充）— §1「路线杠杆证伪」结论不成立

> 本节在 Iter 34 合入后、为 Iter 35 准备时复核 compact.log 发现，**§1 的结论基于"me 走理论最优路"的未经验证假设，实际不成立**。冰鉴阈值修复（§3）独立有效，不受此勘误影响。

### 勘误事实

从 quality-route 桶 6 局 compact.log 的 `A rN MOVE target=SX` 重建 me **实际**路线，一致为：

```
S01 → S02 → S03 → S06 → S08 → S10 → S13 → S14 → S15
```

**不是** §1.2 算出的最优水路 `S01→S02→S04→S05→S09→S10→S13→S14→S15`。me 实际走 S06-S08（MOUNTAIN/BRANCH，最高损耗地形），目的：S06 领 ICE_BOX、S08 领 SHORT_HORSE、沿途 CLAIM_TASK。

### 损耗核算（边权同 §1.2）

| 路线 | 帧 | 鲜度损耗 |
|---|---|---|
| me 实际（山路 S03-S06-S08） | 386 | 24.41 |
| 理论最优（水路 S04-S05-S09） | 322 | 16.98 |
| **Δ（me − 最优）** | **+64** | **+7.43（≈ −13.4 分）** |

+7.43 鲜度损耗几乎等于 quality-route 桶的鲜度 gap 全部。**路线 IS 是杠杆**——me 的贪心瀑布 `_plan` 为领冰鉴/马/任务绕进高损耗山地，全局看不划算。

### 对 Iter 34 结论的修正

- §1「路线选择不是鲜度杠杆」→ **撤销**。路线是杠杆，但不是我算的那条最优路——me 实际走的是次优山路。
- §1.3「静态规划器是错误工具，不开启、后续不再投入 A/B」→ **撤销**。static_planner 的鲜度感知路线重选**可能正是修这个绕山的工具**；Iter 26–28 sim 中性的真正原因待重评（可能是 samples 图无此绕路模式，而非工具无效）。`ENABLE_STATIC_PLANNER` 仍默认关，但**重评开关提上 Iter 35 议程**。
- §3 冰鉴阈值 78→81 → **不受影响，仍有效**（弱优于证明独立于路线；me 走山路时冰鉴在跌 80 前用同样救 1 好果）。
